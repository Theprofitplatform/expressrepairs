# Ad Landing Pages — Design

**Date:** 2026-06-23
**Project:** expressrepairs.com.au (Xpress Phone Repairs, Riverwood Plaza)
**Status:** Implemented (2026-06-23)

## Goal

Stand up a small family of **conversion-first landing pages** for paid ad
campaigns (Google Search, Google Local Services / Maps, and Meta/display
social). Each page is tuned to one ad group and built to produce **phone
calls** — with a callback form as the fallback.

These are deliberately separate from the existing SEO `repairs/` page family:
leaner, call-first, no internal-link clutter, and hidden from organic search so
they never compete with or dilute the SEO pages.

## Decisions (from brainstorming)

| Decision | Choice |
| --- | --- |
| Ad sources | Google Search, Google Local/Maps, Meta + display social |
| Primary CTA | **Call the shop** (tap-to-call). Form is the secondary fallback. |
| Scope | A few hand-crafted pages, one per top service |
| Services | Screen Repair, Battery, Water Damage, + a catch-all "any repair" |
| Hook | Same-day / under an hour · Free diagnostic, no fix no fee · Warranty |
| Tracking | Google Ads conversion tag + Meta Pixel + GA4 |
| Indexing | `noindex, nofollow` + excluded from sitemap |
| URL path | `/go/<service>/` |
| Form | Lean 2-field (name + phone), service pre-filled |

## Pages

Generated by one data-driven route (`src/pages/go/[lp].astro`), mirroring the
existing `repairs/[service].astro` pattern:

| URL | Ad group |
| --- | --- |
| `/go/screen-repair/` | Screen repair |
| `/go/battery/` | Battery replacement |
| `/go/water-damage/` | Water damage |
| `/go/repairs/` | Broad / catch-all "any repair" |

All four map to existing service slugs (`screen`, `battery`, `water-damage`)
plus a catch-all that lists all services. Prices, FAQs and warranty copy are
**derived from the existing `SERVICES` data** in `src/data/repairs.js` so no
figures are duplicated or invented.

## Architecture

```
src/pages/go/[lp].astro        NEW — data-driven landing route (getStaticPaths)
src/data/landing.js            NEW — landing overlay: slug→{adH1, offer, sub, sourceService}
src/data/tracking.js           NEW — tag IDs (placeholders, each optional)
src/components/AdTracking.astro NEW — emits GA4 + Google Ads + Meta Pixel tags
src/components/LandingForm.jsx  NEW — lean 2-field callback island
astro.config.mjs               CHANGED — sitemap filter excludes /go/
tests/landing.test.js          NEW — data integrity + noindex + sitemap-exclusion
```

Untouched: existing `repairs/` SEO pages, homepage, `functions/api/lead.js`.

### Data layer — `src/data/landing.js`

A small overlay keyed by landing slug. Each entry references a source service
(for prices/FAQs/warranty) and supplies ad-specific copy:

```js
export const LANDING_PAGES = [
  {
    slug: 'screen-repair',
    service: 'screen',          // -> SERVICES lookup in repairs.js
    h1: 'Cracked screen? Fixed today.',
    offer: 'Same-day screen repair · Free diagnostic · 6–12 month warranty',
    sub: '...',
  },
  // battery, water-damage, and a `repairs` catch-all (service: null -> all)
];
```

Zod-validated (matching the project convention that `src/data/*` is the
single, validated source of truth). The route throws at build time if a
landing entry references a missing service slug.

## Page anatomy (call-first)

1. **Slim header** — logo + one tap-to-call button. No full nav (clicks don't
   leak off the page).
2. **Hero** — service-specific H1, one-line sub, **large tap-to-call** primary
   CTA + "Prefer a callback?" secondary (anchors to the form). Trust row:
   ⚡ Same-day / under an hour · ✓ Free diagnostic (no fix, no fee) ·
   🛡️ 6–12 month warranty · live **Open now / Today's hours** (reusing the
   `#today-hours` / `#open-status` hooks driven by `PageScripts`). Price-from
   chip from the service data.
3. **Pricing snapshot** — for a single-service page, compact by-brand "from"
   prices reusing `PriceTable`. For the catch-all `/go/repairs/` page (no single
   service), a grid of per-service "from" prices (Screen from $99, Battery from
   $59, …) instead of one brand table.
4. **3-step "how it works"** strip (reused from the service-page pattern).
5. **Short FAQ** — 3–4 objection-busting Q&As from the service's existing FAQ
   list (reusing the accordion behaviour in `PageScripts`).
6. **Callback form** — lean `LandingForm` island: name + phone, service
   pre-filled, posts via `sendLead({ source: 'landing:<slug>', ... })`. On
   success, fires the lead conversion (see Tracking).
7. **Find us** — NAP, today's hours, **Get directions** button (Google Maps
   query from `SITE.mapsQuery`) for Maps/Local traffic.
8. **Sticky mobile call bar** (reusing `MobileCta`) + minimal footer
   (reusing `SiteFooter`).

### Honesty constraints

- `SITE.rating.count` is intentionally `null`; the pages must **not** display a
  fabricated review count or star rating. Trust signals stay truthful:
  same-day, free diagnostic, warranty, all major brands, Riverwood Plaza.
- Prices/turnarounds come only from existing validated data.

## Tracking — `AdTracking.astro` + `src/data/tracking.js`

`tracking.js` holds placeholder IDs, each optional:

```js
export const TRACKING = {
  ga4Id: '',              // 'G-XXXXXXXXXX'
  googleAdsId: '',        // 'AW-XXXXXXXXXX'
  googleAdsCallLabel: '', // conversion label for a call
  googleAdsLeadLabel: '', // conversion label for a form lead
  metaPixelId: '',        // '1234567890'
};
```

`AdTracking.astro` (included only on `/go/` pages) emits each tag **only if its
ID is set** — so nothing loads or errors before the IDs are filled in. A small
client script wires two conversion events:

| Event | Trigger | Fires |
| --- | --- | --- |
| Call | click on `[data-track="call"]` (hero, header, sticky bar, find-us) | Google Ads call conversion · `fbq('track','Contact')` · GA4 `call` |
| Lead | `LandingForm` submit **success** | Google Ads lead conversion · `fbq('track','Lead')` · GA4 `generate_lead` |

Tags load after first paint (non-render-blocking) to protect Core Web Vitals,
since paid traffic is sensitive to load speed.

## SEO / indexing

- `Layout` is rendered with `noindex={true}` → emits
  `<meta name="robots" content="noindex, nofollow">` (prop already supported).
- `astro.config.mjs`: `sitemap({ filter: (page) => !page.includes('/go/') })`
  so the four pages stay out of `sitemap.xml`.
- Canonicals self-reference each `/go/` URL (Layout default) — acceptable since
  the pages are noindexed.

## Error handling

- Lead send failure: `LandingForm` shows a "please call us on (02) 9533 3300"
  message (mirrors existing `sendLead` failure behaviour) — never a false
  success. No lead conversion fires on failure.
- Build-time guard: route throws if a landing entry references an unknown
  service slug, so a typo can't ship a broken page.
- Missing tracking IDs: tags silently no-op (by design).

## Testing (`tests/landing.test.js`, vitest)

- Every `LANDING_PAGES` entry resolves to a real service (or the catch-all).
- Built `/go/*` pages contain `noindex, nofollow`.
- Built `sitemap-*.xml` contains **no** `/go/` URL.
- Each page renders a tap-to-call link to `SITE.phoneHref` and the offer text.
- `AdTracking` emits nothing when IDs are empty (no `gtag`/`fbq` snippet).

## Out of scope (YAGNI)

- A/B testing or query-param personalization.
- Per-device pages (iPhone vs Samsung) — can come later off the same template.
- Cookie-consent banner (AU; no PII beyond name/phone the user submits). Revisit
  if Meta/Google require consent-mode for the target audience.
- Dynamic keyword insertion from ad params.
