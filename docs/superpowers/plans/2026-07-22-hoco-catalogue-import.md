# HOCO Catalogue Import Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Import the HOCO AU supplier catalogue (~3,848 products, currently 99.5% absent) into /shop, merged with the existing DXPOS-synced catalog without being wiped by the next sync.

**Architecture:** The DXPOS sync owns `src/data/products.json` and rewrites it on every run, so HOCO products live in a separate committed file `src/data/hoco-products.json`, generated by a new `scripts/import-hoco.mjs` from a committed catalogue snapshot. The two catalogs are merged (with duplicate suppression) by a pure function in `src/lib/merge-catalogs.js`, consumed by `src/data/products.js` — the single choke point that already feeds every shop page, the search index, and the Google/Meta product feed.

**Tech Stack:** Node ESM scripts (same style as `scripts/sync-products.mjs`), Zod validation, Vitest tests in `repo/tests/`, Astro static build, Cloudflare Pages deploy.

## Background facts (verified 2026-07-22)

- Site catalog: 3,518 products, **100% Mobile Mall SKUs** (via DXPOS). HOCO AU catalogue: 3,848 products, only ~21 name-match the site; the site's 439 hoco-branded items came via Mobile Mall, and ~100–150 of them likely overlap the HOCO sheet by model code.
- `src/data/product-images.json` (10,817 entries) **already contains all 3,848 HOCO image URLs**, keyed by HOCO numeric product ID (e.g. `"8250" → https://www.hoco.com.au/web/image/product.template/8250/image_1024`). Images are hotlinked; `thumbUrl()` in `sync-products.mjs` already rewrites Odoo `image_1024 → image_256` for thumbs.
- Source sheet: `HOCO_Catalogue_with_RRP_2026-07-20.xlsx` (repo root's parent, `C:\Users\sales\claudee\expressrepairs\`), sheet `Catalogue`, columns: Product ID, Product Name, Wholesale Price (AUD), Selling Price / RRP (AUD), Margin (AUD), Margin %, Product URL, Image URL.
- "Some hoco products are missing" (owner suspicion): DXPOS rows are dropped by the sync funnel when they have no image-map hit, wrong grid group, or are archived. After this import, any HOCO-catalogue product not currently live gets imported from the HOCO side regardless, so the merge itself closes that gap. Task 6 reports the funnel numbers anyway.

## Global Constraints

- **The repo is PUBLIC. Never commit wholesale/cost prices.** Only the RRP column may reach any committed file. (Same rule the DXPOS sync applies to cost price.)
- `src/data/products.json` remains DXPOS-only and sync-owned; never hand-edit it, never write HOCO data into it.
- All prices are integer cents, AUD, GST-inclusive (`priceCents`).
- Every product must validate against `productSchema` in `src/data/schema.js` (`image` must be a full URL).
- Tests: Vitest, files in `repo/tests/`, run with `npx vitest run`.
- Working directory for all commands: `C:\Users\sales\claudee\expressrepairs\repo`.
- Deploy: `npm run build` then `wrangler pages deploy dist --project-name expressrepairs --branch main` (the live domain is the Pages project, not a Worker).
- Coordinate with the parallel SEO agent: do not rebase/switch branches under a running deploy; work on a branch, merge to main once.

## Decisions taken (owner can override later, each is one knob)

- **Excluded by default:** professional repair tooling/machines (MaAnt, Sunshine, 2UUL, Mr Yang, Mechanic, Qianli, Relife, Aixun, JCID, TBK brands; OCA/separator/soldering/rework/microscope keywords) and bulk `[PACK n]` trade packs. These are B2B lines in a consumer shop. Knob: `HOCO_EXCLUDE_PATTERNS` in `scripts/import-hoco.mjs`.
- **Price = HOCO RRP column as-is.** Rows with missing/zero RRP are skipped (logged).
- **Duplicates prefer DXPOS** (the shop can actually ring those up): a HOCO import row is dropped when its exact fixed name already exists in DXPOS data, or (hoco-branded items) when its model-code set is a subset of a single existing hoco product's codes.
- **Shipping copy unchanged** ("dispatched in 1-2 business days"). Owner should confirm HOCO order lead time; if longer, that's a copy change, not a code change.
- **Known ceiling (accepted):** COCO case-plus-glass bundles ("… /w Tempered Glass Protector") get filed under Screen Protection by the shared `fixCategory` glass keyword. Either category is defensible for a bundle; if the owner wants them under Cases & Covers, widen the `case with|with case` guard in `catalog-fixes.mjs` — one regex, not part of this plan.

```python
# scripts/extract-hoco-catalogue.py — HOCO xlsx -> src/data/hoco-catalogue.json
# Usage: python scripts/extract-hoco-catalogue.py "../HOCO_Catalogue_with_RRP_2026-07-20.xlsx"
# The repo is public: ONLY the RRP column is exported. Wholesale never leaves this script.
import json, sys, pathlib
import openpyxl

src = sys.argv[1]
ws = openpyxl.load_workbook(src, read_only=True)["Catalogue"]
rows = ws.iter_rows(values_only=True)
header = next(rows)
assert header[:2] == ("Product ID", "Product Name"), f"unexpected sheet layout: {header}"

out, skipped = [], 0
for r in rows:
    pid, name, _wholesale, rrp = r[0], r[1], r[2], r[3]
    image = r[7]
    if not pid or not name or not isinstance(rrp, (int, float)) or rrp <= 0:
        skipped += 1
        continue
    out.append({
        "id": int(pid),
        "name": str(name).strip(),
        "rrpCents": round(float(rrp) * 100),
        "image": str(image or "").strip(),
    })

dest = pathlib.Path(__file__).parent.parent / "src" / "data" / "hoco-catalogue.json"
dest.write_text(json.dumps(out, indent=1) + "\n", encoding="utf-8")
print(f"{len(out)} products written, {skipped} skipped (no id/name/RRP)")
```

```js
// scripts/import-hoco.mjs — transform the committed HOCO catalogue snapshot
// (src/data/hoco-catalogue.json, RRP only) into src/data/hoco-products.json
// for the /shop pages. DXPOS knows nothing about these products; they are
// merged with the DXPOS sync at build time by src/lib/merge-catalogs.js, so
// a POS re-sync can never wipe them. Images are hotlinked from hoco.com.au.
//
// Re-run after refreshing the snapshot:
//   python scripts/extract-hoco-catalogue.py "<new xlsx>"
//   node scripts/import-hoco.mjs
import { readFileSync, writeFileSync } from 'node:fs';
import { fileURLToPath, pathToFileURL } from 'node:url';
import { applyCatalogFixes } from './catalog-fixes.mjs';
import { thumbUrl } from './sync-products.mjs';

// B2B lines that must not appear in a consumer shop: repair machines/tooling
// brands and bulk trade packs. Owner: add a pattern to hide a class, or
// delete one to start selling it.
export const HOCO_EXCLUDE_PATTERNS = [
  /\b(MaAnt|Sunshine|2UUL|Mr Yang|Mechanic|Qianli|Relife|Aixun|JCID|TBK)\b/i, // tool brands
  /\b(OCA|separator|soldering|rework station|microscope|glue remover|reballing|test board|programmer)\b/i,
  /\[PACK \d+\]/i, // bulk trade packs
  /\bBull W\b/i, // bulk-glass trade brand
];

// Keyword -> existing site category. Order matters: protectors before cases
// (a "case with glass" bundle is filed as a case by the negative lookahead
// in catalog-fixes' fixCategory, which runs after this and also handles the
// AirPods/tablet/watch case splits).
const CATEGORY_RULES = [
  [/tempered glass|screen protector|privacy glass|matte glass|hydrogel|camera (lens |)?(protector|glass|guard)/i, 'Screen Protection'],
  [/\bcase\b|\bcover\b|\bfolio\b|\bpouch\b|ring stand/i, 'Cases & Covers'],
  [/\bcable\b|charger|charging|power bank|powerbank|\badapter\b|\badaptor\b|\bdock\b|car charge/i, 'Cables & Charging'],
  [/earbud|earphone|headphone|headset|speaker|microphone|\bmic\b/i, 'Audio'],
  [/holder|mount|\bstand\b|tripod|selfie stick/i, 'Mounts & Holders'],
];
export const hocoCategory = (name) =>
  CATEGORY_RULES.find(([re]) => re.test(name))?.[1] || 'Accessories';

// Pure transform: catalogue snapshot rows -> productSchema-shaped entries.
// applyCatalogFixes gives us the shared name repairs, brand inference, the
// cases/protectors category refinements, and exact-name dedupe for free.
export function transformHoco(rows) {
  return applyCatalogFixes(
    rows
      .filter(
        (r) =>
          r.rrpCents > 0 &&
          r.image &&
          !HOCO_EXCLUDE_PATTERNS.some((p) => p.test(r.name)),
      )
      .map((r) => ({
        id: `H-${r.id}`,
        name: r.name,
        category: hocoCategory(r.name),
        brand: '', // fixBrand infers from the name (hoco./COCO/Hanman/platform)
        priceCents: r.rrpCents,
        image: r.image,
        thumb: thumbUrl(r.image),
        inStock: true,
        sku: String(r.id),
      })),
  );
}

if (process.argv[1] && import.meta.url === pathToFileURL(process.argv[1]).href) {
  const src = fileURLToPath(new URL('../src/data/hoco-catalogue.json', import.meta.url));
  const rows = JSON.parse(readFileSync(src, 'utf8'));
  const products = transformHoco(rows);
  const excluded = rows.filter((r) => HOCO_EXCLUDE_PATTERNS.some((p) => p.test(r.name)));
  console.log(
    `hoco funnel: catalogue=${rows.length} excluded-trade=${excluded.length} -> importable=${products.length}`,
  );
  if (products.length === 0 || products.length > 6000) {
    console.error(`refusing to publish ${products.length} products — check the snapshot/patterns.`);
    process.exit(1);
  }
  const out = fileURLToPath(new URL('../src/data/hoco-products.json', import.meta.url));
  writeFileSync(out, JSON.stringify(products, null, 2) + '\n');
  console.log(`Wrote ${products.length} HOCO products.`);
}
```

```js
// src/lib/merge-catalogs.js — combine the DXPOS-synced catalog with the HOCO
// catalogue import at build time. DXPOS wins every duplicate (those products
// exist in the POS and can be rung up in-store); HOCO fills the rest.
// ponytail: name + hoco-model-code dedupe only — cross-supplier fuzzy name
// matching is a rabbit hole; the worst miss is one product listed twice.

// Letter+digit tokens like G9 / J170 / DCA71 / UA43A. Device tokens (A27,
// S26) match too — that's why the check below requires the FULL code set of
// the import row to be covered by a single existing product, not a global pool.
export const modelCodes = (name) =>
  new Set(name.match(/\b[A-Z]{1,3}\d{1,3}[A-Z]{0,2}\b/g) || []);

export function mergeCatalogs(dxpos, hoco) {
  const names = new Set(dxpos.map((p) => p.name));
  const hocoCodeSets = dxpos
    .filter((p) => /hoco\./i.test(p.name))
    .map((p) => modelCodes(p.name))
    .filter((s) => s.size);
  const covered = (codes) =>
    codes.size && hocoCodeSets.some((s) => [...codes].every((c) => s.has(c)));
  const fresh = hoco.filter(
    (p) => !names.has(p.name) && !(/hoco\./i.test(p.name) && covered(modelCodes(p.name))),
  );
  return [...dxpos, ...fresh];
}
```
